mb_strtolower, image size bypass and zip uploading

上傳處對圖片的尺寸做出了要求，此時有一種方法來繞過。將下列程式碼拼接到文章最後即可繞過 getimagesize 的檢測。

檔名也經過了過濾，結合後面的程式碼可知需要上傳一個 zip 壓縮文件。使用 Latin Capital Letter I with Dot Above 也就是 İ 可以繞過 stristr 並且在 mb_strtolower 之後變成小寫字母 i。此時上傳壓縮檔案變為可能，而 example.php 中提供瞭解壓的業務。只要生成一個惡意的壓縮檔案使其解壓後變成一個 PHP 檔案即可寫入 shell。配合目錄穿越即可將 shell 寫到可訪問的 Web 目錄下。

imagecreatefrompng bypass

example.php 中對檔案進行了檢測和裁剪，因此直接對圖片進行修改無法透過檢測從而使最後的檔案被刪除。因此此時需要生成一個可以透過檢測的圖片馬。

https://www.idontplaydarts.com/2012/06/encoding-web-shells-in-png-idat-chunks/

https://github.com/huntergregal/PNG-IDAT-Payload-Generator/

使用參考的 Repo 中的程式碼可以生成一個長度為 25 的任意 PHP payload 的正方形圖片，只需要將自帶的 payload 經過 Raw Deflate 之後再 Inflate 即可。這裡直接放上最終的 PoC。

tiny traffic

篩選流量包中的 HTTP 流量，可以發現一部分響應中有 Brotli 的壓縮流量。

將傳輸的二進位制檔案匯出，在 Linux 中透過 yum install brotli 安裝 brotli 工具，使用 brotli --stdout --decompress /root/data/extract.bin 來解碼內容。在 /test 路由的傳輸內容解碼後可以得到如下 ProtoBuf3 結構。
