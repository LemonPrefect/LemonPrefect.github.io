友情連結layout折騰記錄

LemonPrefect
2020-02-11
Daily
BlogModify, CSS, Layout, Script

之前 blog 上的友情連結板塊一直依靠於一個簡單的 widget 實現，一開始甚至是一個完全不能容納任何細節的列表式清單。後來照著其他 widget 的寫法，稍微修改了一下 layout 實現了簡潔的詳細資訊展示，但是 BUG 也隨之而來。在不同縮放比例下，內容會溢位容器，顯得極不協調。再者，單獨的 widget 始終不是長久之計，於是折騰了一個頁面。

初期探索

Icarus 這個主題使用 ejs 來生成頁面，因此，我大致整理了一些細節。

幾種標籤的意義

  • <% ‘指令碼’ 標籤，用於流程控制，無輸出。

  • <%_ 刪除其前面的空格符

  • <%= 輸出資料到模板（輸出是轉義 HTML 標籤）

  • <%- 輸出非轉義的資料到模板

  • <%# 註釋標籤，不執行、不輸出內容

  • <%% 輸出字串 ‘<%’

  • %> 一般結束標籤

  • -%> 刪除緊隨其後的換行符

  • _%> 將結束標籤後面的空格符刪除

_config.yml 與程式碼的連結

在 layout 的程式碼中，使用 theme 這一個物件可以訪問整個 theme/_config.yml 中的所有配置。而使用物件的訪問方法可對其逐層遞進， theme.widgets 可以訪問到各 widget 的配置。但這時候的 theme.widgets 實際上已經變成了一個可遍歷的陣列，其形式有點類似於 Python 中的字典。而這些 key 對應的 value 又是一個物件，當你嘗試把它輸出，會得到 [Object object] 這樣的結果。因此，為了訪問到單獨的 friend 這一層，就得轉換兩次。

layout 與頁面渲染的連結

要將頁面模板與獨立頁面連結在一起，使 Hexo 在生成頁面的時候按照所寫模板生成，只需要將模板放到 root/to/theme/path/layout 下。然後新建一個普通頁面。在頂部新增 layout: layoutName 一行然後儲存。 注意空格！！
