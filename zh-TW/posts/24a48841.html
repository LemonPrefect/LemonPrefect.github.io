日常賽事解題記

LemonPrefect
2020-04-12

最近零零散散在各種從各種渠道參加的賽事上寫了些水題，一直在糾結到底要不要記。不過既然都做了，感覺過程中也有不少騷操作，就稍微記一記。正好也記上這些比賽裡沒寫出來的題目的復現。

Dawg CTF 2020

Free WIFI Part 1

附件給了個 free-wifi.pcapng，匯入到 WireShark 裡略微分析，可以發現一個請求。

嘗試訪問 staff.html，並根據上述請求構造一個 POST 請求，就能在頁面底端拿到 flag。

The Lady is a Smuggler

這題一開始給了個頁面，下面有一段加密，但其實解密那段得出來的 flag 是無效的。真實的 flag 在頁面的原始碼中。

Tracking

這題頁面感覺跟前面那個一樣，首先看一手原始碼。找到一段 JavaScript。

放到 Console 裡面執行就得到了 flag。

Deep CTF

Oh JS!

JSFuck

這題開門見山地給了個登入的 form，但其實檢視頁面原始碼可以看到一段 JSFuck，把它掐頭去尾放到 Console 跑一下可以發現線索。

於是直接跳過登入的步驟，訪問 .../4d4932602a75414640946d38ea6fefbf.php 得到 flag。

Magic Word!

Regexp

這題給出了頁面的原始碼和 Try to reach get_mad_and_give_flag() 的提示。

既然是 preg_replace()，之前在 SQL 注入的題目裡面遇到過，只需要夾著寫一遍就行了。於是構造 ?magic_word=d3d33p3p 得到 flag。

Nothing is Impossible

這題給的是一個可以執行 php 程式碼的框，於是果斷試了一手 system()，發現仍然可以使用。根據題目提示，flag 在 /tmp/flag.php。於是直接使用 od 構造了 system("od -A d -c /tmp/flag.php")，得到結果。

整理之後得到 flag。

Did You Got Trolled?

這題我成功地被繞了進去，他給了好多動物的叫聲，我還以為是動物園。還有那個假裝可以注入的地方，也是很好笑。其實真正的要點在找到兩個 key。在學長的提示下，我在 .../css/clean-blog.css 下找到了 Key1。其實他是有給提示的，只有那一個頁面引入了同樣的兩份 CSS，且重複的那份是沒有壓縮的。

接下來的那一份就得掃目錄了，是在 .../robots.txt 裡面有個提示，寫著 Disallow /deep.php。訪問這個頁面可以得到一個提示。

很容易知道這是一個檔案包含，然後再看一手頁面原始碼，可以找到 <!--Creds in /home/ubuntu/key2.txt --> 的註釋。於是構造 ?page=/home/ubuntu/key2.txt 得到 Key2。

回到 .../post.html 提交兩個 Key，可以得到 flag。

Greetings!

SSTI

根據題目提示，傳遞一個 GET 引數 ?name=you 可以得到 Hello you 的反饋。於是試了一手 {{7 * 7}} 得到了 Hello 49 的反饋，可以確定是 SSTI 的考點，然後發現了 tornado。順勢試了下 ?name={{__import__("os").popen("ls").read()}} 找到了 flag.txt。於是再構造 cat flag.txt 得到 flag。

Hack MEEEE

Ruby

上來先看了一手這題原始碼，找到了 <!-- This application is made using ruby! <3 --> 的提示，然後當時觸及到了知識盲區，就沒管。其實翻了一波手冊再查了一下之後發現很簡單。頁面請求傳遞了兩個 POST 引數。這裡有個坑，method 要放在前面，不然 POST 本身的換行符加上之後可能不會有正確結果返回。（報錯在頁面原始碼的註釋中可以看到）

按照 Ruby 的手冊，使用 method=methods 和 method=private_methods 可以檢視當前類下包含的方法。於是找到了可以利用的反引號。手冊參考 結合上 instance_eval 這個方法就能達成目的。

因此，構造 method=instance_eval&message=`ls` 的 POST 請求引數，可以得到 flag.txt 位於當前目錄下的線索，於是只需要再構造 cat flag.txt 即可得到 flag。

TG:Hack

Shop

題目一開始，給了 bank 和 store 的兩個頁面，我們知道“襯衫的價格是九鎊十五便士”。啊，不是，是二十五美元。這題嘛，其實不能按照一般思路去想，一開始我在 bank 頁面試了好久都加不上錢，然後就只能去 store 頁面找。每買一件東西都會傳遞兩個 POST 引數，分別指代物品類別和錢數。所以，把錢數想辦法變成負數就可以從商店“索要”金錢了。但是還有一個點，使用已經存在的物品 ID 不可信，所以就隨便整個其他的 ID 就行。這裡還有一個坑，POST 引數最後的換行符會有影響（所以 HackBar 不行），我使用某線上的 POST 工具就成功達成目的了。最後就可以買到 flag。

Redux

這題，說實話我其實沒太看懂。首先隨便填一填他給的表，然後提交，Console 裡就能看到一條訊息 The form has been submitted! Look at the Redux store!。我一開始也想知道這個 Redux store 指的是哪裡，但是找了幾下沒找到。看到這條訊息是來自 form.js 的，於是就順手點開看了看。

於是就找到了 flag。

AUCTF

Miyazaki Trivia

開啟頁面有 Find this special file. 的提示，於是找到 .../robots.txt。開啟之後又是新的提示 VIDEO GAME TRIVIA: What is the adage of Byrgenwerth scholars? MAKE a GET request to this page with a header named 'answer' to submit your answer.，查了一下之後得到答案，於是構造 ?answer=Fear the Old Blood 得到 flag。

Quick Maths

這題先隨便算幾個數，發現可以正常算出結果。（不太記得了，寫的時候好像算了 7 * 7 啥的，得到的都是49）於是順手構造了個 statement=system("ls") 然後發現有返回結果。於是就安排上 statement=system("cat ./index.php") 就拿到了 flag。需要注意的是，這裡的請求包需要用 Burp Suite 傳送才能得到返回的結果。

gg no re

這題一開始給了個 authentication.js，略微分析之後可以發現是 Base64，取出關鍵資訊 TWFrZSBhIEdFVCByZXF1ZXN0IHRvIC9oaWRkZW4vbmV4dHN0ZXAucGhw再解碼，得到 Make a GET request to /hidden/nextstep.php。按照提示進行，在 Header 中可以找到下一步線索，ROT13: Znxr n CBFG erdhrfg gb /ncv/svany. CuC。使用工具解密一下，得到 Make a POST request to /api/ final. PhP，接著按照提示 Send a request with the flag variable set 進行，構造 ?flag=1 即可得到 flag。

API madness

按照提示訪問 .../static/help 可以看到有三個 API。先嚐試一下 login，這裡有個坑，要用 curl 發請求才能成功 curl -X POST -H "Content-Type: application/json" -d '{"username":"lemon","password":"password"}' host/api/login，而且響應的時間很長很長，返回的資料是一個報錯的頁面，裡面包含了一個不一樣的介面 .../api/login_check。curl 訪問之後得到了一個為 null 的 token。看起來像是假的一樣，但是就像愛情，你不去嘗試怎麼知道結果呢？於是按照 API 介面接著往前衝，構造 curl -X POST -H "Content-Type: application/json" -d '{"dir":"/","token":"null"}' host/api/ftp/dir，返回結果中看到了 flag.txt。於是構造 curl -X POST -H "Content-Type: application/json" -d '{"file":"/flag.txt","token":"null"}' host/api/ftp/get_file 得到結果。

取出 YXVjdGZ7MHdAc3BfNnJvSzNOX0B1dGh9Cg== 之後 Base64 解碼即得 flag。

GWCTF 2019

我有一個數據庫

這題上來就說了資料庫，但是開啟頁面發現沒得入口，只有兩行字。於是嘗試找了一下特殊檔案，找到了 robots.txt。檔案裡給了個 phpinfo.php，但是開啟之後沒找到啥利用點。但是由於自己搭環境的時候出了些問題，於是去查了一波，就事先知道了這題有個 phpmyadmin，如果真的打比賽估計找不到這個點。不過他給的 phpmyadmin 很奇怪，不需要登入就能操作資料庫。於是就去查了一波相關的知識點，找到了 CVE-2018-12613。

於是照著教程，構造 .../index.php?target=db_events.php%3f/../../../../../flag 就能夠包含到在根目錄的 flag。

枯燥的抽獎

頁面給了十個隨機字母，檢視網頁原始碼可以發現一個 check.php，訪問之後可以拿到隨機數生成的方法，按照 MRCTF-Ezaudit 的套路用 php_mt_seed 這個工具算一波隨機數種子，然後再用 PHP 生成後面十位就行了。這裡踩了一個坑，工具的版本很重要，我之前使用的那個在這題算不出來而且能算出來的也不會告知 PHP 的版本號。至於生成適合工具使用的引數格式嘛，我寫了一段 PHP 程式碼。

如果前面的步驟都沒有錯，算出來的隨機數種子應該是 951373997。放到給出的原始碼中然後解除 10 位的限制，可以得到二十位的字串 oS668h2wc93iAcrwCuZQ。提交到頁面上即可得到 flag。

(後面的題目環境搭好了 題目好像很頂 讓我慢慢寫一波)

Sharky CTF 2020

XXExternalXX

上來點 Show stored data 探索一波，注意到了位址列的 ?xml=data.xml 察覺到可能是 XXE。隨便填充之後在報錯裡發現了 Trying to get property 'data' of non-object，於是根據題目描述順手構造一波。

這裡因為是個 GET 引數，所以得想辦法給構造的 payload 一個直鏈才能成功，達成之後可得 flag。

Logs In ! Part 1

這題千萬不要被標題迷惑，訪問頁面之後可以在左下角找到 Controller 的頁面，順手訪問一波之後可以看到兩個路由，訪問其中一個 /e48e13207341b6bffb7fb1622282247b/debug 可得 flag。

Containment Forever

這題的兩條 flag 沒辦法開啟，但是有其他記錄可以訪問。根據地址猜測應該是要自己找到這個類似於 ID 的東西。於是嘗試隨便改一波，發現報錯了且找到了 mongoose 的字樣。在大佬的指引下找到了 參考，於是開始把頁面上原來的 ID 拆開。

然後發現隨機數是固定的，也就是說只需要時間戳和 ID。時間戳只需要從兩條記錄上找然後轉換一下，很容易得到。

至於 ID 的話只需要根據已經有的 ID 在附近列舉一下就可以了。於是寫段程式碼生成一下。

將得到的幾個結果逐一嘗試可以得到包含 flag 的兩個記錄的地址。

將得到的 flag 拼接在一起，得到 flag。

Aqua World

開啟頁面，發現導航有個點不動的地方，審查元素訪問到 .../admin-query?flag=flag。頁面提示需要修改 netloc 為本地。在各種修改 header 之後看到了題目給的 hint：WTF this PYTHON version is deprecated!!!。於是順著線索找到了 CVE-2019-9636 並根據其中給出的例子構造（Authorization 的 header 是題目本身要求的）。
